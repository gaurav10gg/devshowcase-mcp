# dev-showcase-mcp

> MCP server for Dev_showcase API

Generated by [Hatch](https://hatch.dev) from your API.

## Setup

```bash
npm install
cp .env.example .env       # fill in your credentials
npm run build
```

## Connect to Claude Desktop

After `npm run build`, the server entry is at `dist/index.js`. Add the following
block to your Claude Desktop config (`~/Library/Application Support/Claude/claude_desktop_config.json`
on macOS, `%APPDATA%/Claude/claude_desktop_config.json` on Windows):

```json
{
  "mcpServers": {
    "dev-showcase-4d43": {
      "command": "node",
      "args": ["${PWD}/dist/index.js"],
      "env": {
        "BASE_URL": "your-base_url-here"
      }
    }
  }
}
```

Restart Claude Desktop. The 22 tools below should appear in the tool list.

## Tools

- `upload_project_image` — Upload a project image to Supabase storage via multipart/form-data. Requires an image file as input and returns the public URL of the uploaded image along with a success message.
- `upload_avatar` — Upload a user avatar image to Supabase storage via multipart form data. Requires the avatar file as a form field and returns the public URL of the uploaded image upon success.
- `authorize_oauth` — Initiates the OAuth authorization flow by redirecting to the frontend OAuth connect page. Requires a redirect URI to return the user to after authorization and a state parameter for CSRF protection and session continuity.
- `exchange_oauth_token` — Exchanges a Supabase access token for an OAuth authorization code by completing the OAuth callback flow. Requires a valid access token, redirect URI, and state parameter to prevent CSRF attacks. Returns a redirect URL containing the authorization code for the client to follow.
- `exchange_oauth_token` — Exchange an OAuth authorization code for an access token. Requires a valid client ID, client secret, and authorization code obtained from the OAuth authorization flow. Returns a bearer access token upon successful exchange.
- `delete_comment` — Deletes a comment by its ID. Only the owner of the comment is permitted to perform this action. Returns a confirmation message upon successful deletion.
- `create_project` — Creates a new project on behalf of the authenticated user via the GPT action endpoint. Accepts project metadata including title, descriptions, tags, and optional links to a live demo or GitHub repository. Returns the created project object along with a success indicator.
- `delete_project` — Delete a project by its unique ID. Permanently removes the specified project and returns a confirmation message upon success.
- `sync_user` — Syncs a Supabase user into the Neon users table by creating or updating the user record with the provided id, email, and name. All three fields are required to perform the sync operation.
- `get_current_user_profile` — Retrieves the profile of the currently authenticated user. Returns account details associated with the active session or token. No additional parameters are required.
- `list_users` — Retrieves a list of all users in the system. Returns the full collection of user records available to the caller.
- `update_user_profile` — Updates the authenticated user's profile information. All fields are optional — supply only the fields you wish to change, such as bio, avatar URL, social links, or username.
- `delete_user` — Deletes a user by their unique ID. This operation is permanent and cannot be undone. Returns a confirmation message upon successful deletion.
- `create_project` — Creates a new project with the provided details. The title field is required; all other fields such as description, image, tags, and external links are optional. Returns the newly created project upon success.
- `list_projects` — Retrieves all projects, optionally reflecting liked status when authentication context is provided. Returns a collection of project records available in the system.
- `get_my_projects` — Retrieves all projects belonging to the currently logged-in user. Returns a list of projects associated with the authenticated account.
- `get_project` — Retrieves a single project by its unique ID. Returns the full project details for the specified project.
- `update_project` — Updates an existing project by its ID with the provided fields. The title and short_desc fields are required; all other fields (live URL, tags, image, GitHub link, full description) are optional. Returns the updated project on success.
- `like_project` — Like a project by its ID, toggling or registering the current user's like. Returns the updated like count and the liked status after the operation.
- `unlike_project` — Remove a like from a project on behalf of the authenticated user. Requires authentication. Returns the updated like status and total like count for the project.
- `add_project_comment` — Add a comment to a specified project by providing the project ID and comment text. The request requires authentication and a non-empty text body. Returns the created comment on success.
- `get_project_comments` — Retrieves all comments associated with a specific project. Requires a valid project ID as a path parameter.

## Notes

- The generated code uses **stdio** transport — Claude Desktop spawns it as a subprocess. To run as an HTTP/SSE server instead, swap `StdioServerTransport` in `src/index.ts`.
- Tool definitions live in `src/tools.ts`. Edit field descriptions, add validators, or remove tools you don't want exposed.
- Auth is injected by `src/lib/http.ts`. The auth type is fixed to `none` — change it there if your API uses a different scheme.
